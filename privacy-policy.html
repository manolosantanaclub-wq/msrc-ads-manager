MSRC Ads Manager · Internal Tool

Privacy Policy

Effective date: 23 March 2026  ·  Last updated: 23 March 2026

Summary: This is an internal tool used exclusively by Racquets Club Marbella S.L. to manage its own Meta advertising campaigns. No personal data from third parties is collected, processed, or shared.

1. About this application

MSRC Ads Manager is a private, internal application developed and operated exclusively by Racquets Club Marbella S.L. (Manolo Santana Racquets Club), Marbella, Spain. It connects to the Meta Marketing API to create, manage, and analyse the club's own advertising campaigns on Facebook and Instagram. This application is not available to the public.

2. Data collected

This application does not collect personal data from any individual other than the authorised operator of the club's own Meta Business account.

  • No user registration or sign-up is required.
  • No cookies or tracking technologies are deployed on any public-facing interface.
  • Campaign data accessed via the Meta API contains only aggregated, anonymised metrics provided by Meta.
  • No personally identifiable information (PII) of end users or club members is accessed, stored, or processed.

3. Meta API access and permissions

This application uses a Meta User Access Token with the following permissions, solely for managing Racquets Club Marbella's own advertising:

  • ads_read — to read campaign performance data.
  • ads_management — to create and modify campaigns.
  • pages_manage_ads — to manage ads linked to the club's Facebook Page.
  • business_management — to access Business Manager settings.

All access tokens are stored locally on the authorised operator's device and are never transmitted to any third-party server.

4. Data sharing

Racquets Club Marbella S.L. does not sell, rent, trade, or share any data obtained through this application with third parties.

5. Data retention

Campaign reports are retained internally according to applicable Spanish and EU regulations, including GDPR and Ley Orgánica de Protección de Datos (LOPD-GDD).

6. Security

Access is restricted to authorised personnel. API credentials are managed securely on local, password-protected devices.
